## 1. Conceitos Fundamentais

### O que é Cloud Computing?

**Cloud Computing** é a entrega de serviços de computação (servidores, armazenamento, bancos de dados, rede, software) pela internet ("a nuvem") com pagamento conforme o uso, em vez de possuir e manter infraestrutura física própria.

Imagine que em vez de comprar um carro (servidor físico), você usa Uber (cloud) - paga apenas quando precisa, sem se preocupar com manutenção, seguro ou estacionamento.

### Modelos de Serviço Cloud

#### Infrastructure as a Service (IaaS)
**IaaS** fornece infraestrutura virtualizada sob demanda:
- **Você gerencia**: OS, runtime, aplicações, dados
- **Provedor gerencia**: Virtualização, servidores, armazenamento, rede
- **Exemplos AWS**: EC2, VPC, EBS
- **Analogia**: Alugar um apartamento vazio - você traz os móveis

#### Platform as a Service (PaaS)
**PaaS** fornece plataforma de desenvolvimento e deployment:
- **Você gerencia**: Aplicações e dados
- **Provedor gerencia**: Runtime, OS, middleware, infraestrutura
- **Exemplos AWS**: Elastic Beanstalk, Lambda, RDS
- **Analogia**: Apartamento mobiliado - você só traz suas coisas pessoais

#### Software as a Service (SaaS)
**SaaS** fornece aplicações completas pela internet:
- **Você gerencia**: Apenas dados e configurações
- **Provedor gerencia**: Tudo (aplicação, plataforma, infraestrutura)
- **Exemplos**: Gmail, Office 365, Salesforce
- **Analogia**: Hotel - você só usa, tudo está pronto

### Por que AWS?

**Amazon Web Services (AWS)** é o provedor de cloud líder mundial:
- **Market Share**: ~33% do mercado global
- **Pioneirismo**: Primeiro grande provedor (2006)
- **Amplitude**: 200+ serviços
- **Presença Global**: 84 Availability Zones em 26 regiões
- **Confiabilidade**: Usado por Netflix, Airbnb, NASA

### Infraestrutura Global AWS

#### Regiões (Regions)
**Regiões** são áreas geográficas com múltiplos data centers:
- **Isolamento**: Cada região é completamente independente
- **Latência**: Escolha região próxima aos usuários
- **Compliance**: Algumas regiões atendem regulamentações específicas
- **Exemplos**: us-east-1 (N. Virginia), sa-east-1 (São Paulo)

#### Availability Zones (AZs)
**AZs** são data centers isolados dentro de uma região:
- **Redundância**: Cada região tem 2+ AZs
- **Conectividade**: Interconectadas com baixa latência
- **Falhas**: Isoladas para evitar falhas em cascata
- **Alta Disponibilidade**: Distribua recursos entre AZs

#### Edge Locations
**Edge Locations** são pontos de presença para CDN:
- **CloudFront**: Rede de distribuição de conteúdo
- **Latência**: Conteúdo mais próximo dos usuários
- **Global**: 400+ edge locations mundialmente

### Serviços Fundamentais AWS

#### Amazon EC2 (Elastic Compute Cloud)
**EC2** fornece capacidade computacional redimensionável:

**Tipos de Instância:**
- **General Purpose**: t3, m5 (uso geral)
- **Compute Optimized**: c5 (CPU intensivo)
- **Memory Optimized**: r5, x1 (memória intensiva)
- **Storage Optimized**: i3, d2 (I/O intensivo)

**Modelos de Preço:**
- **On-Demand**: Pague por hora/segundo
- **Reserved**: Desconto por compromisso de 1-3 anos
- **Spot**: Até 90% desconto, pode ser interrompido
- **Dedicated**: Hardware dedicado

```bash
# Exemplo: Lançar instância EC2
aws ec2 run-instances \
  --image-id ami-0abcdef1234567890 \
  --instance-type t3.micro \
  --key-name minha-chave \
  --security-group-ids sg-12345678 \
  --subnet-id subnet-12345678
```

#### Amazon VPC (Virtual Private Cloud)
**VPC** é sua rede virtual privada na AWS:

**Componentes:**
- **Subnets**: Segmentos de rede (públicas/privadas)
- **Internet Gateway**: Acesso à internet
- **NAT Gateway**: Internet para subnets privadas
- **Route Tables**: Roteamento de tráfego
- **Security Groups**: Firewall de instância
- **NACLs**: Firewall de subnet

```yaml
# Exemplo: VPC com Terraform
resource "aws_vpc" "main" {
  cidr_block           = "10.0.0.0/16"
  enable_dns_hostnames = true
  enable_dns_support   = true
  
  tags = {
    Name = "main-vpc"
  }
}

resource "aws_subnet" "public" {
  vpc_id                  = aws_vpc.main.id
  cidr_block              = "10.0.1.0/24"
  availability_zone       = "us-east-1a"
  map_public_ip_on_launch = true
  
  tags = {
    Name = "public-subnet"
  }
}
```

#### Amazon S3 (Simple Storage Service)
**S3** é armazenamento de objetos altamente escalável:

**Características:**
- **Durabilidade**: 99.999999999% (11 9's)
- **Escalabilidade**: Virtualmente ilimitado
- **Classes de Storage**: Standard, IA, Glacier, Deep Archive
- **Versionamento**: Múltiplas versões do mesmo objeto
- **Encryption**: Em trânsito e em repouso

```bash
# Comandos S3 CLI
aws s3 mb s3://meu-bucket-unico
aws s3 cp arquivo.txt s3://meu-bucket/
aws s3 sync ./pasta s3://meu-bucket/pasta/
aws s3 ls s3://meu-bucket/
```

#### Amazon RDS (Relational Database Service)
**RDS** é serviço gerenciado de banco de dados:

**Engines Suportadas:**
- MySQL, PostgreSQL, MariaDB
- Oracle, SQL Server
- Amazon Aurora (MySQL/PostgreSQL compatível)

**Benefícios:**
- **Gerenciamento**: Patches, backups automáticos
- **Escalabilidade**: Vertical e horizontal (read replicas)
- **Alta Disponibilidade**: Multi-AZ deployment
- **Segurança**: Encryption, VPC, IAM integration

### Segurança e Identidade

#### AWS IAM (Identity and Access Management)
**IAM** controla acesso aos recursos AWS:

**Componentes:**
- **Users**: Identidades individuais
- **Groups**: Coleções de usuários
- **Roles**: Identidades assumíveis
- **Policies**: Documentos de permissão JSON

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject"
      ],
      "Resource": "arn:aws:s3:::meu-bucket/*"
    }
  ]
}
```

#### Princípios de Segurança
- **Least Privilege**: Mínimas permissões necessárias
- **Defense in Depth**: Múltiplas camadas de segurança
- **Shared Responsibility**: AWS cuida da infraestrutura, você dos dados
- **Encryption**: Sempre que possível, em trânsito e repouso

### Monitoramento e Logging

#### Amazon CloudWatch
**CloudWatch** monitora recursos e aplicações:
- **Métricas**: CPU, memória, rede, custom metrics
- **Logs**: Centralização e análise de logs
- **Alarms**: Notificações baseadas em métricas
- **Dashboards**: Visualização customizada

#### AWS CloudTrail
**CloudTrail** registra atividades da conta AWS:
- **API Calls**: Quem fez o quê, quando
- **Compliance**: Auditoria e conformidade
- **Security**: Detecção de atividades suspeitas
- **Troubleshooting**: Rastreamento de mudanças

### Modelo de Responsabilidade Compartilhada

#### Responsabilidade da AWS
- **Infraestrutura física**: Data centers, hardware
- **Hypervisor**: Virtualização e isolamento
- **Rede**: Infraestrutura de rede global
- **Serviços gerenciados**: RDS, Lambda, S3

#### Responsabilidade do Cliente
- **Dados**: Classificação e proteção
- **Identidade**: IAM users, groups, roles
- **Aplicações**: Código e configurações
- **OS**: Patches e atualizações (EC2)
- **Rede**: Security groups, NACLs, firewall

### Precificação AWS

#### Fatores de Custo
- **Compute**: Tipo e tamanho da instância
- **Storage**: Quantidade e tipo de armazenamento
- **Data Transfer**: Tráfego de saída (ingress gratuito)
- **Requests**: Número de requisições (APIs, Lambda)

#### Otimização de Custos
- **Right Sizing**: Instâncias adequadas ao workload
- **Reserved Instances**: Compromisso de longo prazo
- **Spot Instances**: Workloads tolerantes a interrupção
- **Auto Scaling**: Ajuste automático de capacidade
- **Cost Explorer**: Análise de gastos

## 2. Questões de Múltipla Escolha

### Questão 1
Qual é a principal diferença entre IaaS, PaaS e SaaS?

a) IaaS é mais caro que PaaS e SaaS
b) Diferem no nível de gerenciamento que o provedor assume
c) IaaS só funciona na AWS, PaaS no Google Cloud, SaaS no Azure
d) São sinônimos, não há diferença

**Gabarito: B**

**Justificativa:** Os modelos diferem no nível de abstração e responsabilidade. IaaS fornece infraestrutura virtualizada (você gerencia OS, runtime, aplicações), PaaS fornece plataforma de desenvolvimento (você gerencia apenas aplicações), e SaaS fornece aplicações completas (você apenas usa). Quanto mais alto o nível, menos você gerencia e mais o provedor assume.

### Questão 2
Por que é recomendado distribuir recursos entre múltiplas Availability Zones?

a) Para reduzir custos de transferência de dados
b) Para garantir alta disponibilidade e tolerância a falhas
c) Para melhorar a performance de rede
d) Para atender regulamentações de compliance

**Gabarito: B**

**Justificativa:** Availability Zones são data centers fisicamente separados dentro de uma região, projetados para serem isolados de falhas. Distribuir recursos entre AZs garante que se uma AZ falhar (energia, rede, desastre), sua aplicação continue funcionando nas outras AZs. É a base da alta disponibilidade na AWS.

### Questão 3
No modelo de Responsabilidade Compartilhada da AWS, quem é responsável por aplicar patches de segurança no sistema operacional de uma instância EC2?

a) AWS é sempre responsável por patches de OS
b) Cliente é responsável por patches de OS em EC2
c) Responsabilidade é compartilhada entre AWS e cliente
d) Depende do tipo de instância EC2

**Gabarito: B**

**Justificativa:** Em EC2 (IaaS), o cliente tem controle total sobre o sistema operacional e é responsável por patches, atualizações e configurações de segurança do OS. A AWS cuida da infraestrutura física e do hypervisor, mas tudo "dentro" da instância é responsabilidade do cliente. Em serviços gerenciados como RDS, a AWS cuida dos patches.
