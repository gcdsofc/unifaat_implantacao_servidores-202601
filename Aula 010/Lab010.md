# Lab 10: Conceitos de Infraestrutura em Nuvem e AWS

**Disciplina:** Implementação de servidor e nuvem (cloud)  
**Aula:** 10 - Conceitos de Infraestrutura em Nuvem e AWS  
**Objetivo:** Instalar e configurar a AWS CLI no ambiente WSL/Linux e executar comandos básicos para inspecionar serviços essenciais (IAM e S3), simulando a gestão remota da Nuvem.

## Objetivo do Lab

Ao final deste laboratório, você será capaz de:
- Instalar e configurar AWS CLI no WSL/Linux
- Autenticar e gerenciar credenciais AWS
- Inspecionar serviços IAM (Identity and Access Management)
- Gerenciar buckets e objetos no S3
- Consultar instâncias EC2 via linha de comando
- Compreender modelos de cloud (IaaS, PaaS, SaaS)

### Pré-requisitos

* AConta AWS Ativa: Necessário para as Seções 4, 5 e 6 (Cloud real).

* Ambiente WSL2 (Linux): Com distribuição Ubuntu ou similar instalada e atualizada.

* Docker Desktop: Instalado no Windows, com a opção "Use the WSL 2 based engine" ativada e integração habilitada para a sua distribuição Linux.

* Permissões de Administrador (Sudo): No WSL para instalação de pacotes.

* Conexão com a Internet: Para download do binário da AWS CLI e da imagem Docker do LocalStack.

## Cenário: Gerenciamento Básico da AWS via Linha de Comando (WSL)

Usaremos o terminal WSL/Linux como nossa ferramenta principal para interagir com a AWS, reforçando a importância do ambiente Linux no DevOps.

## Passo a passo: Criar uma conta na AWS
Antes de instalar a AWS CLI, o aluno deve criar uma conta AWS válida. Siga estes passos para criar a conta e preparar o acesso:

1. Acesse o site da AWS: https://aws.amazon.com
2. Clique em "Create an AWS Account" ou "Criar conta gratuita".
3. Informe um e-mail válido, nome completo e senha segura.
4. Preencha os dados de cobrança com um cartão de crédito ou débito válido.
5. Verifique seu telefone via SMS ou chamada, seguindo as instruções na tela.
6. Escolha o plano de suporte gratuito (Basic Support).
7. Após criação, faça login no Console da AWS.
8. No Console, acesse o menu do usuário e selecione "My Security Credentials" ou "Credenciais de Segurança".
9. Crie um usuário IAM com permissões administrativas ou use um usuário existente com acesso a IAM, S3 e EC2.
10. Gere uma Access Key ID e Secret Access Key para o usuário IAM e salve essas informações em local seguro.

> Observação: Não use a conta root para operações diárias. Crie um usuário IAM com permissões adequadas para proteger a conta.

## Seção 1: Preparação do Ambiente e Instalação da AWS CLI
A AWS CLI (Command Line Interface) é o cliente que permite gerenciar os serviços AWS remotamente e será necessária para todas as seções seguintes.

1.  Crie o diretório de trabalho do laboratório.
```bash
mkdir -p ~/aulas_lab/aula010
cd ~/aulas_lab/aula010
```

2.  **Instalação (Ambiente Linux - WSL):** Use o `curl` para baixar e instalar o pacote da AWS CLI (método comum para ambientes Linux/WSL).
```bash
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install
```

3.  **Verificação:** Confirme se a instalação foi bem-sucedida.
```bash
aws --version
```

## Seção 2: Instalação e Uso do LocalStack via Docker
LocalStack é uma ferramenta que emula serviços AWS localmente, permitindo testar a AWS CLI sem usar uma conta real. Usaremos Docker para executar o LocalStack.

1.  Inicie o LocalStack usando Docker:
```bash
docker run --rm -d \
  -e SERVICES=s3,iam,ec2 \
  -e DEFAULT_REGION=sa-east-1 \
  -p 4566:4566 \
  localstack/localstack:0.14.3
```

> Observação: A versão mais recente do LocalStack requer licença. Use a versão 0.14.3 que ainda funciona gratuitamente.

2.  Verifique se o LocalStack está rodando corretamente testando a conectividade:
```bash
curl -s http://localhost:4566/_localstack/health | grep services
```

3.  Para usar o AWS CLI com o LocalStack, adicione o endpoint local (`http://localhost:4566`) aos comandos:
```bash
aws --endpoint-url=http://localhost:4566 s3 ls
```

> Observação: use o LocalStack para testes locais e a AWS real quando precisar de recursos na nuvem.

## Seção 3: Configuração de Credenciais AWS
Configure suas credenciais para usar tanto a AWS real quanto o LocalStack.

1.  **Configuração para AWS real:** configure suas credenciais e região:
```bash
aws configure
# [ENTER] AWS Access Key ID: **********
# [ENTER] AWS Secret Access Key: **********
# [ENTER] Default region name: sa-east-1
# [ENTER] Default output format: json
```

2.  **Configuração para LocalStack:** caso use LocalStack, crie um perfil separado com credenciais dummy e use endpoint local.
```bash
aws configure --profile localstack
# AWS Access Key ID: test
# AWS Secret Access Key: test
# Default region name: sa-east-1
# Default output format: json
```

3.  Verifique se a configuração foi salva corretamente:
```bash
aws configure list
```

   Deve mostrar algo como:
   | Name | Value | Type | Location |
   |------|-------|------|----------|
   | profile | <not set> | None | None |
   | access_key | ****************XXXX | shared-credentials-file |
   | secret_key | ****************XXXX | shared-credentials-file |
   | region | sa-east-1 | config-file | ~/.aws/config |

> Observação: para comandos com LocalStack, use **apenas** `--endpoint-url=http://localhost:4566` sem combinar com `--profile localstack`. O perfil localstack configurado nesta seção é para referência, mas não é necessário ao usar `--endpoint-url`.

## Seção 4: Inspeção do IAM (Identidade e Acesso) - AWS real
Vamos inspecionar seu próprio usuário para provar que a autenticação funcionou com a AWS real.

1.  **Verificar Identidade com AWS real:** Obtenha detalhes do usuário IAM ativo (Root ou IAM User).
```bash
aws sts get-caller-identity
```
*(**Resultado Esperado:** Uma saída JSON com o `UserId`, `Account` e `Arn` deve ser exibida, comprovando a autenticação.)*

2.  **Listar Usuários IAM (AWS real):** Liste os primeiros 10 usuários IAM na sua conta.
```bash
aws iam list-users --max-items 10
```

> **Nota sobre LocalStack:** O comando `aws sts get-caller-identity` não é compatível com a versão 0.14.3 do LocalStack. Use LocalStack apenas para os testes de S3 e EC2 nesta atividade.

## Seção 5: Gerenciamento Básico do S3 (Armazenamento) - AWS / LocalStack
O S3 (Simple Storage Service) é o serviço de armazenamento de objetos. Vamos listar e inspecionar *buckets*.

1.  **Listar Buckets (AWS real):** Liste todos os *buckets* S3 na sua conta.
```bash
aws s3 ls
```
*(Se não houver nenhum, a lista estará vazia.)*

2.  **Listar Buckets (LocalStack):** Liste buckets no LocalStack.
```bash
aws --endpoint-url=http://localhost:4566 s3 ls
```

3.  **Criar um Bucket (AWS real - Opcional):** Crie um *bucket* de teste. **Atenção: O nome do *bucket* deve ser globalmente único e em letras minúsculas.**
```bash
aws s3 mb s3://lab10-devops-unifaat
```

4.  **Criar um Bucket (LocalStack - Opcional):** Crie um *bucket* de teste no LocalStack (nomes podem ser repetidos localmente).
```bash
aws --endpoint-url=http://localhost:4566 s3 mb s3://lab10-devops-unifaat
```

5.  **Listar Conteúdo (AWS real):** Se o *bucket* foi criado, liste o conteúdo (inicialmente vazio).
```bash
aws s3 ls s3://lab10-devops-unifaat
```

6.  **Listar Conteúdo (LocalStack):** Se o *bucket* foi criado no LocalStack, liste o conteúdo.
```bash
aws --endpoint-url=http://localhost:4566 s3 ls s3://lab10-devops-unifaat
```

## Seção 6: Gerenciamento de Instâncias EC2 (Infraestrutura) - AWS / LocalStack
Vamos criar e inspecionar instâncias EC2 simulando o gerenciamento de infraestrutura.

1.  **Descrever Instâncias (AWS real):** Liste o ID, o estado e o tipo de todas as instâncias EC2 na região configurada.
```bash
aws ec2 describe-instances \
--query 'Reservations[].Instances[].[InstanceId, State.Name, InstanceType]'
```
*(**Resultado Esperado:** Uma lista dos seus EC2s (se houverem) no formato de matriz será exibida.)*

2.  **Criar Instância EC2 (AWS real - Opcional):** Crie uma instância EC2 usando uma AMI válida da sua região.
```bash
aws ec2 run-instances \
  --image-id ami-0123456789abcdef0 \
  --count 1 \
  --instance-type t2.micro \
  --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=lab10-instance}]'
```

3.  **Listar Instâncias novamente (AWS real):** Verifique a criação da instância.
```bash
aws ec2 describe-instances \
--query 'Reservations[].Instances[].[InstanceId, State.Name, InstanceType, Tags[?Key==`Name`].Value[]]'
```

4.  **Criar Instância EC2 (LocalStack):** Crie uma instância EC2 local usando LocalStack.
```bash
aws --endpoint-url=http://localhost:4566 ec2 run-instances \
  --image-id ami-0c55b159cbfafe1f0 \
  --count 1 \
  --instance-type t2.micro \
  --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=lab10-localstack-instance}]' \
  --output text
```

5.  **Descrever Instâncias (LocalStack):** Liste instâncias EC2 no LocalStack.
```bash
aws --endpoint-url=http://localhost:4566 ec2 describe-instances \
--query 'Reservations[].Instances[].[InstanceId, State.Name, InstanceType]'
```

## Seção 6.5: Conexão SSH com Instâncias EC2 - AWS / LocalStack

Após criar uma instância EC2, você pode conectar-se a ela via SSH para gerenciamento remoto.

### Conexão SSH com AWS real

1. **Obter o IP público da instância:**
```bash
aws ec2 describe-instances \
--query 'Reservations[].Instances[].[InstanceId, PublicIpAddress, KeyName]' --output text
```

2. **Conectar via SSH (substitua `key.pem` pela sua chave privada e `IP_PUBLICO` pelo IP obtido):**
```bash
ssh -i key.pem ec2-user@IP_PUBLICO
```

> **Nota:** Certifique-se de que o Security Group permite tráfego SSH (porta 22) e que a chave privada tem permissões 400 (`chmod 400 key.pem`).

### Conexão SSH com LocalStack

LocalStack emula serviços EC2, mas não executa instâncias reais de VM. Portanto, **não é possível conectar via SSH a uma instância criada no LocalStack**, pois ela não existe fisicamente. Use LocalStack apenas para testes de comandos CLI, não para conexões reais.

## Seção 7: Limpeza Final

1.  **Remover o Bucket (AWS real - Se criado):** Se você criou o *bucket* de teste, remova-o (deve estar vazio).
```bash
aws s3 rb s3://lab10-devops-unifaat
```

2.  **Remover o Bucket (LocalStack - Se criado):** Se você criou o *bucket* no LocalStack, remova-o.
```bash
aws --endpoint-url=http://localhost:4566 s3 rb s3://lab10-devops-unifaat
```

3.  **Remover Instância EC2 (AWS real - Se criada):** Liste a instância e termine-a.
```bash
aws ec2 describe-instances \
--query 'Reservations[].Instances[].[InstanceId, State.Name, InstanceType]' --output text
INSTANCE_ID=$(aws ec2 describe-instances \
--query 'Reservations[].Instances[].InstanceId' --output text | awk '{print $1}')
echo "$INSTANCE_ID"
aws ec2 terminate-instances --instance-ids "$INSTANCE_ID"
```

4.  **Remover Instância EC2 (LocalStack - Se criada):** Liste a instância e termine-a no LocalStack.
```bash
aws --endpoint-url=http://localhost:4566 ec2 describe-instances \
--query 'Reservations[].Instances[].[InstanceId, State.Name, InstanceType]' --output text
INSTANCE_ID=$(aws --endpoint-url=http://localhost:4566 ec2 describe-instances \
--query 'Reservations[].Instances[].InstanceId' --output text | awk '{print $1}')
echo "$INSTANCE_ID"
aws --endpoint-url=http://localhost:4566 ec2 terminate-instances --instance-ids "$INSTANCE_ID"
```

5.  **Limpar Credenciais (Opcional):** As credenciais são armazenadas no arquivo `~/.aws/credentials`. Para removê-las, você pode apagar o arquivo.
```bash
rm -r ~/.aws
```

6.  **Parar o LocalStack Container (se estiver em execução):**
```bash
docker stop $(docker ps -q --filter "ancestor=localstack/localstack:0.14.3")
```

7. Apagar pastas
```bash
cd ~
rm -rf ~/aulas_lab/aula010
```
